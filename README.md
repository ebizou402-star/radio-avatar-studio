# EBIZO TALK Avatar Studio

Spotifyで配信中の `EBIZO TALK` 向けに作った、音声連動の二人収録アバター画面です。ブラウザで開いて、マイクまたは音源ファイルの音量に合わせて左右のキャラクターが反応します。

ホスト版・ゲスト版の手順は [EBIZO TALK 取扱説明書 PDF](output/pdf/EBIZO_TALK_Manual_JA.pdf) にまとめています。

## 起動

```bash
python3 -m http.server 5173
```

開くURL:

```text
http://127.0.0.1:5173/
```

OBSなどに載せる場合は、ブラウザソースで以下を使うと操作パネルなしの画面になります。

```text
http://127.0.0.1:5173/?stage=1
```

## できること

- 左右のマイク入力をそれぞれ別キャラクターに割り当て
- ステレオ音源ファイルの左チャンネル/右チャンネルで別々に反応
- モノラル音声では自動、左、右、二人モードを切り替え
- ライブ入力をブラウザ内で録音し、録音後にファイルとして保存
- 人物を消した生成背景を固定し、人物の頭・体・口だけを音量に合わせて動かす
- 感度、動き、余韻を調整

## ライブ収録

1. 左に自分のマイク、右に相手の通話音声を割り当てます。
2. `マイク` を押して、二人の声で左右のキャラクターが反応するか確認します。
3. `録音開始` を押して収録します。
4. `録音停止` を押すと、録音ファイルのリンクが出ます。

相手が日本にいても、Zoom、Discord、Google Meetなどの通話音声を右入力に入れられればライブ収録できます。通話音声を別入力として扱うには、Macの仮想オーディオ設定が必要になる場合があります。

## 遠隔で二人同時収録

同じアプリを二人で開き、接続コードをメッセージなどで送り合うと、WebRTCで直接つながります。録音は左が自分、右が相手の音声です。

`http://127.0.0.1:5173/` は自分のPC専用のURLなので、相手に送っても相手側では開けません。相手も同じファイルを自分のPCで起動するか、この静的アプリを公開URLに置いて同じページを開く必要があります。

無料で安全寄りに公開する場合は、GitHub Pagesの公開リポジトリ運用に限定します。`lhr.life`、`localhost.run`、`ngrok` などの一時トンネルURLはセキュリティソフトに弾かれやすいため、アプリ側でも遠隔操作をブロックします。公開後は以下のようなURLになります。

```text
https://<github-user>.github.io/<repository-name>/
https://<github-user>.github.io/<repository-name>/?guest=1
```

ホスト側:

1. `招待` を押します。
2. `送るコード` を `コピー` して相手に送ります。
3. 相手から返ってきた回答コードを `相手コード` に貼ります。
4. `適用` を押します。
5. `接続中` になったら `録音開始` を押します。

ゲスト側:

1. ホストから届いた招待コードを `相手コード` に貼ります。
2. `回答` を押します。
3. `送るコード` を `コピー` してホストに返します。
4. `接続中` になったら、必要なら自分側でもバックアップ録音します。

携帯だけの相手には、同じ公開URLの末尾に `?guest=1` を付けたページを開いてもらいます。ゲスト画面は相手のマイク送信用に絞った表示です。スマホのマイク許可は基本的にHTTPSページで必要になるため、日本の相手に使ってもらう場合は、この静的アプリをHTTPSの公開URLに置く必要があります。

遠隔接続にはWebRTCの接続確認用に無料の公開STUNサーバーを使います。音声データをアプリ側のサーバーへ保存・アップロードする処理や、課金APIは入れていません。安定収録のため、二人ともイヤホン/ヘッドホン推奨です。

## セキュリティー

- 公開用URLは `github.io` のHTTPS URLを使います。
- 一時トンネルURLでは遠隔ボタンが無効になります。
- ブラウザのCSPで外部JavaScript、フォーム送信、埋め込み、不要な外部通信を制限しています。
- 招待コードと回答コードは接続情報なので、相手だけに送ってください。アプリ内のコード欄は録音ファイル作成後と切断時に自動削除されます。チャットへ送ったコードは収録後に手動で削除してください。
- 詳細は [SECURITY.md](SECURITY.md) を確認してください。
