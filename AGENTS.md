# EBIZO TALK Development Roles

このリポジトリの変更は、必ず次の三段階で進める。

## 1. 制作部隊

- ユーザーの要望を実装する。
- 既存の画面、音声処理、セキュリティ方針を維持する。
- 外部課金が必要なサービスは、実行前にユーザーの承認を得る。
- 実装後に `dist/index.html` と配布ZIPを生成する。

## 2. 修正部隊

制作完了後、制作時の思い込みを引き継がず、次を独立して確認する。

- JavaScriptの構文と起動処理
- 画像・CSS・JavaScriptの参照切れ
- マイク許可、録音開始・停止、音声連動
- 遠隔接続、招待コード、自動削除
- CSP、許可URL、一時トンネル拒否
- 単一ファイル版と配布ZIPの整合性
- デスクトップと携帯幅での表示崩れ

不具合を見つけた場合は修正し、全検査を最初から再実行する。新機能は追加しない。

## 3. 最終確認

`tools/quality_gate.mjs` がすべて成功し、必要な画面確認も完了した場合だけ公開可能とする。確認できない項目がある場合は合格扱いにせず、理由をユーザーへ明示する。

## 4. マネタイズ部門

公開可能になった制作物を使い、収益化の調査、商品設計、価格テスト、KPI管理、営業準備を進める。

- 最初は無料で実行できる施策を優先する。
- 有料広告、課金サービス、決済口座接続、サブスクリプション開始は、実行直前にユーザーの承認を得る。
- スポンサー候補へのメール、DM、フォーム送信は、文面と送信先を示してユーザーの承認を得る。
- パスワード、決済情報、税務情報、非公開の個人情報をリポジトリへ保存しない。
- 収益化のためにアプリを変更する場合は、制作部隊へ戻し、修正部隊の最終確認を再度通す。

運用資料は `monetization/` で管理する。
